# Sample output: News Anchor (Studio)

Real, unedited reference output for the [`news-anchor`](../../../presets/news-anchor.json)
preset — generated by running the published script verbatim:

```bash
python3 examples/design_voice.py --preset presets/news-anchor.json --seeds 8160
```

Three previews come back from seed `8160`. **`cand-03-seed8160-3.mp3` is the keeper** —
the warm, grounded, unflappable take. The other two are the same seed's other previews,
included so you can hear the range a single seed produces.

If your own run sounds flat or like a generic voiceover and *these* don't, the difference
is in your params or prompt handling — see the Troubleshooting section in
[`SKILL.md`](../../../SKILL.md).

## What's in each candidate

Every candidate ships with its full set of reproducibility sidecars:

| File | Contents |
|------|----------|
| `.mp3` | the generated audio |
| `.prompt` | the exact `voice_description` (scene prompt) used |
| `.text` | the exact mundane audition text used |
| `.seed` | the seed (8160) |
| `.params` | model, guidance_scale, enhance_prompt, loudness, reference_audio |
| `.voice_id` | the `generated_voice_id` ElevenLabs returned for that preview |

`_manifest.json` lists all three candidates with durations.

### About the `.voice_id` files

These are ElevenLabs **design-preview** IDs. They are tied to the account that generated
them and they **expire** — they are not credentials and are not usable on another account
(any API call needs that account's key). They're included here only for completeness. The
**portable** way to reproduce this voice on any machine is the recipe in
[`news-anchor.json`](../../../presets/news-anchor.json): same model + seed + prompt + text +
settings yields the same voice. The seed is the fingerprint, not the `voice_id`.
